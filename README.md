# Telegram English Tutor Bot

Телеграм-бот для адаптивного изучения английского языка на основе интеллектуального механизма подбора сложности.

Проект реализует ТЗ из файла: уровни A1–C2, уроки, быстрые вопросы, аудио-уроки, адаптивное тестирование уровня и сбор статистики. Логика организована по слоям и старается следовать принципам SOLID.

## Стек

- Python 3.11
- [aiogram 3.x](https://docs.aiogram.dev/) — Telegram Bot API
- PostgreSQL + SQLAlchemy 2.x (async)
- Alembic — миграции
- loguru + стандартный `logging` — логирование
- Docker + docker-compose

## Основные возможности

- Урок-тест (10 вопросов: grammar + vocabulary).
- Аудио-урок (10 вопросов: listening + speaking).
- Быстрый вопрос (grammar / vocabulary / listening / speaking).
- Оценка произношения (обработчик голосовых сообщений).
- Два варианта тестирования уровня:
  - Адаптивный тест с вопросами.
  - Анализ текста пользователя.
- Статистика пользователя: streak по дням, пройденные уроки, % правильных ответов, уровень и др.
- База заранее сгенерированных заданий, выбор только "свежих" задач.
- Прогресс-бар по вопросам в уроках и тестах.
- Сообщения о генерации заданий ("Генерирую вопрос..." и т.п.).

## Интеллектуальная часть (ML)

В папке `app/core/ml` находятся интерфейсы и базовые заглушки:
- `question_generator.py` — генерация заданий по уровням и темам.
- `difficulty_regressor.py` — регрессионная модель подбора сложности.
- `emotion_analyzer.py` — анализ эмоционального состояния (заглушка).
- `pronunciation_evaluator.py` — интерфейс оценки произношения.
- `tts_engine.py` — интерфейс TTS для заданий на аудирование.

По умолчанию используется простая шаблонная генерация вопросов (быстро и бесплатно).  
Вы можете заменить реализацию на интеграцию с локальной LLM (Ollama / LLaMA / BLOOM и т.п.), не меняя остальной код — достаточно реализовать нужные интерфейсы.

## Архитектура слоёв

- `app/main.py` — точка входа, конфигурация бота, роутеры, middlewares.
- `app/config.py` — настройки проекта (Pydantic BaseSettings).
- `app/logging_config.py` — настройка логирования.
- `app/handlers/` — слой представления (обработка Telegram-апдейтов).
- `app/keyboards/` — Reply и Inline клавиатуры.
- `app/states.py` — FSM-состояния уроков/тестов.
- `app/db/` — база данных, сессии, модели и репозитории.
- `app/core/services/` — бизнес-логика (уроки, тесты, статистика и т.д.).
- `app/core/ml/` — "умный" слой, работа с моделями и персонализацией.
- `app/utils/` — вспомогательные утилиты (прогресс-бар, кэш и пр.).

Такое разделение облегчает сопровождение и расширение, а также позволяет подменять реализации (DI) и соблюдать принципы SOLID.

## Быстрый старт

1. Скопируйте файл окружения:

```bash
cp .env.example .env
```

2. Укажите токен бота и при необходимости параметры БД в `.env`:

```env
BOT_TOKEN=ВАШ_ТОКЕН_ТУТ
```

3. Соберите и запустите контейнеры:

```bash
docker-compose up --build
```

По умолчанию контейнер бота выполнит `alembic upgrade head`, затем запустит Telegram-бота.

## Миграции

Создание новой миграции (изнутри контейнера или локально, если установлен alembic):

```bash
alembic revision --autogenerate -m "new changes"
alembic upgrade head
```

## Ограничения и доработка

- Модули ML (эмоции, регрессия сложности, TTS, оценка произношения) реализованы в виде заглушек с понятными интерфейсами.
- Для production-использования их нужно заменить на реальные модели/сервисы (например, локальные LLM и ASR/TTS).
- Логика предзагрузки и буферизации заданий реализована упрощённо (на основе шаблонов), но структура сервисов рассчитана на пакетную генерацию и кэширование.
